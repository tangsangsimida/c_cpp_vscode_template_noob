# C/C++ 菜鸟学习模板

该项目是一个专为初学者设计的现代 C/C++ 学习模板，支持 Windows、Linux 和 macOS 平台。它集成了自动化构建、代码规范检查和内存安全调试等功能。

## 核心特性

1. 自动项目命名：系统会自动获取当前工作目录名称作为项目名。
2. 零维护构建：采用递归检索模式，自动识别所有子目录下的源代码文件和头文件。
3. 现代标准支持：默认配置为 C++23 和 C11，支持动态切换语言版本。
4. 代码质量管理：集成 Clang-Format 自动美化和 Clang-Tidy 静态代码检查。
5. 内存安全调试：内置 AddressSanitizer (ASan) 预设，能够精准捕捉内存泄露和越界错误。
6. 静态链接优化：针对 Windows/MinGW 环境默认启用静态链接，生成的程序可独立运行。

## 快速开始

1. 环境依赖：确保系统已安装 CMake、Ninja 和 GCC/Clang 编译器，并已加入系统环境变量。
2. 配置项目：在 VS Code 中按下 Ctrl+Shift+P，输入并运行 "CMake: Delete Cache and Reconfigure"。
3. 选择预设：在底部的 CMake 状态栏选择 "default" (默认调试) 或 "debug-asan" (内存检测)。
4. 构建与运行：按 Ctrl+Shift+B 执行构建，按 F5 启动调试。

关于动态标准切换、静态检查、内存分析以及多文件开发的详细操作说明，请参阅：[docs/使用指南](docs/usage.md)。

## 文件结构

- include/ ：存放库文件和公共头文件（支持子文件夹）。
- src/ ：存放源代码文件（支持子文件夹），已内置 main.cpp 入口。
- docs/ ：存放项目相关的学习笔记与技术文档。
- .clang-format ：代码美化规则设置。
- .clang-tidy ：代码质量检查规则设置。
- CMakePresets.json ：构建预设配置文件。

## 工具化部署建议

如果您正在编写工具来快速生成新项目：
1. 直接克隆整个目录并重命名文件夹即可。
2. 模板会自动提取新文件夹名作为项目名，无需修改 CMakeLists.txt。
3. 调试器会自动追踪最新的构建路径，无需手动调整路径。
